# ระบบบันทึกการขนย้ายมูลไก่ออกจากฟาร์มและรับเข้ากอง (Chicken Manure Transfer Tracking System)

ระบบเว็บแอปพลิเคชันที่พัฒนาด้วย **Laravel 11**, **MariaDB**, **Bootstrap 5** และ **Laravel Breeze** สำหรับใช้ตรวจบันทึกข้อมูลรถขนย้ายมูลไก่ออกจากฟาร์ม (ขาออก) และตรวจรับเข้ากองปลายทาง (ขารับ) เพื่อให้ข้อมูลตรวจสอบย้อนหลังได้อย่างถูกต้อง ปลอดภัย และไร้ความขัดแย้งเชิงเวลา (Concurrency & Race Condition Proof)

---

## 🌟 คุณสมบัติเด่นของระบบ (Key Features)

1. **รองรับการใช้งานบนมือถือ 100% (Mobile-First Layout)**
   - ทุกหน้าจอได้รับการออกแบบให้แสดงผลได้สวยงามบนสมาร์ตโฟน แท็บเล็ต และเดสก์ท็อป
   - ปรับปุ่มกด อินพุต และกล่องอัปโหลดรูปภาพให้เหมาะกับนิ้วมือของผู้ใช้ในฟาร์ม
2. **ระบบการแบ่งบทบาทหน้าที่อย่างรัดกุม (Role-Based Authorization)**
   - แบ่งสิทธิ์ผู้ใช้เป็น 4 ระดับ: ผู้ดูแลระบบ (Admin), เจ้าหน้าที่ฟาร์ม (Farm Staff), เจ้าหน้าที่กอง (Pile Staff), และ ผู้บริหาร (Viewer)
3. **การทำงานที่มีเสถียรภาพและปลอดภัยต่อการกดเบิ้ล (Transaction Safety)**
   - การรันรหัสเลขที่รายการ `MF-YYYYMMDD-XXX` และขั้นตอนการรับเข้ากอง ใช้ระบบ **Database Transaction** ร่วมกับ **Row-Level Lock (`lockForUpdate`)** เพื่อป้องกันข้อมูลซ้ำซ้อนหรือข้อผิดพลาด Concurrency
4. **ระบบออกรายงานและการส่งออกข้อมูลที่ครบครัน**
   - **Excel (CSV)**: ส่งออกข้อมูลพร้อมกำหนด UTF-8 BOM ทำให้รองรับภาษาไทยใน Microsoft Excel 100% ไม่เจอปัญหารหัสภาษาเพี้ยน
   - **PDF (A4 Landscape Print View)**: ออกแบบหน้าพิมพ์รายงานแผ่นงานแนวนอนอย่างเป็นระเบียบ เรียบร้อย พร้อมลายเซ็นผู้ตรวจสอบ เปิดหน้าสั่งพิมพ์ของเบราว์เซอร์ (`window.print()`) ทันที ทำให้ใช้ฟอนต์สารบัญภาษาไทย (Google Kanit) ได้อย่างคมชัดโดยไม่มีปัญหาสระลอย
5. **ประวัติและภาพหลักฐานประกอบ**
   - แนบภาพหลักฐานขาออกจากฟาร์ม และขารับเข้ากอง พร้อมระบบพรีวิวรูปภาพทันทีบนหน้าเว็บ
   - ตรวจจับขนาดภาพ (ห้ามเกิน 5MB) และสกุลไฟล์ที่ถูกต้องตามมาตรฐานความปลอดภัย

## 🛠️ โครงสร้างเทคโนโลยี (Tech Stack)

- **Backend**: Laravel 11 (PHP 8.2+)
- **Database**: MariaDB 10.11 (ผ่าน Docker Container พอร์ต `3308`)
- **Frontend**: Blade Template + Bootstrap 5 (CSS HSL Forest Green Agriculture Theme) + Bootstrap Icons
- **Authentication**: Laravel Breeze (Blade Stack)
- **Timezone**: `Asia/Bangkok` (บันทึกเวลาตรงตามประเทศไทย)
- **Locale**: `th` (แสดงผลภาษาไทยทั้งหมด)

## 👤 สิทธิ์ผู้ใช้งานสำหรับเข้าสู่ระบบ (Default Credentials)

ในระบบการทดสอบได้ติดตั้งผู้ใช้ตัวอย่างไว้ทั้งหมด 4 ระดับ (รหัสผ่านเริ่มต้นคือ `password` สำหรับทุกบัญชี):

| บทบาทหน้าสิทธิ์ | อีเมลบัญชีผู้ใช้ | สิทธิ์การเข้าใช้งานเมนู |
| --- | --- | --- |
| **ผู้ดูแลระบบ (Admin)** | `admin@example.com` | เข้าได้ทุกเมนู, จัดการข้อมูล CRUD ของฟาร์ม กอง รถ และผู้ใช้ได้ทั้งหมด |
| **เจ้าหน้าที่ฟาร์ม (Farm Staff)** | `farm@example.com` | เข้าบันทึกรายการรถขาออก (Transfer Out) และดูรายงาน/Dashboard |
| **เจ้าหน้าที่กอง (Pile Staff)** | `pile@example.com` | เข้าตรวจรับรถมูลไก่เข้ากองปลายทาง (Transfer In) และดูรายงาน/Dashboard |
| **ผู้บริหาร / ผู้ดูรายงาน (Viewer)** | `viewer@example.com` | เข้าดูหน้า Dashboard สรุปผล, ดูประวัติย้อนหลัง และส่งออก Excel/PDF เท่านั้น |

## ⚙️ ขั้นตอนการติดตั้งและการรันใช้งานจริง (Installation Guide)

### 1. การตั้งค่าสภาพแวดล้อม (Environment Config)
โคลนโปรเจกต์นี้และสร้างไฟล์ `.env` ตรวจสอบความถูกต้องของพารามิเตอร์ฐานข้อมูลให้เชื่อมต่อกับ MariaDB (พอร์ต `3308` บน Local):

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3308
DB_DATABASE=chicken_manure
DB_USERNAME=itadmin
DB_PASSWORD=Noipalee@09
```

### 2. ติดตั้งไลบรารีและแพ็กเกจที่จำเป็น
รันคำสั่งเพื่อโหลดข้อมูลฝั่ง Composer (PHP):

```bash
composer install
```

### 3. ทำการ Migrations และ Seed ข้อมูลพื้นฐาน
ติดตั้งตารางฐานข้อมูลและข้อมูลตั้งต้น (ฟาร์ม, กองมูลไก่ 1-22, รายการทะเบียนรถ, และผู้ใช้จำลอง):

```bash
php artisan migrate:fresh --seed
```

### 4. สร้าง Storage Link สำหรับจัดการรูปภาพ
ระบบต้องการลิงก์เพื่อให้ผู้ใช้ภายนอกเข้าถึงรูปภาพหลักฐานที่อัปโหลดเข้าสู่โฟลเดอร์ `storage/app/public/transfers` ได้อย่างถูกต้อง:

```bash
php artisan storage:link
```

### 5. รันเซิร์ฟเวอร์จำลองการทำงาน
เปิดใช้งานเซิร์ฟเวอร์พัฒนาของ Laravel:

```bash
php artisan serve
```
จากนั้นเข้าใช้งานผ่านเว็บเบราว์เซอร์ที่: [http://127.0.0.1:8000](http://127.0.0.1:8000)

โปรเจกต์มาพร้อมกับชุดการทดสอบการจำลองเหตุการณ์จริง (Feature tests) ซึ่งครอบคลุม:

- การบันทึกขารับเข้ากองปลายทางพร้อมจัดการอัปเดตความถูกต้องและล็อกข้อมูล (Race-Condition-Proof)

```bash
php artisan test
```
